Tables Persisted

  • vinpac1: Stores the raw data that is uploaded via CSV.
  • vinpacCleaned: Stores the cleaned data.
  • Machine_Status_Change: Stores the status change along with the duration it has been in that status for each machine.
  • MachineStoppageChange: Groups and sums the data from Machine_Status_Change based on the machine and its status. (used in visualisation)
  • MachineDetailsFillerStoppage: Stores the stoppage details – the number of status changed and the duration of that state for all machines in each Filler stopped states.
  • MachDetFillerStoppageEachDay: Groups the data from MachineDetailsFillerStoppage based on the date for all machines in each Filler stopped states.
  • MachStoppageforFillerAllDays: Stores the data from MachDetFillerStoppageEachDay in a time-series type after imputing values for the missing days as 0.(used in dashboard)
  • MBA_Filler_Changed: Stores the data prepared for association analysis.
  • MBA_All_Results: Stores all the rules obtained after performing association analysis using apriori with a confidence of 50%.
  • MBA_Filler_All_Results: Stores all the rules for Filler machine’s stopped states which is filtered from the rules in the MBA_All_Results.
  • State4_State5_Results: Stores the rules for all machines’ Faulted and Unallocated stoppage states.
  • MBA_Final_Results: Stores the final association results after performing backtracking to find the main responsible machine causing the Filler to stop.
  • UpdatedDetails: Stores the file upload date along with the date range in the file.
